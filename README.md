# متجر بطاقات Google Play - Starter

وصف مختصر:
هذا مشروع بدء (starter) لمتجر إلكتروني لبيع بطاقات Google Play. يتكوّن من جزء خلفي (Node + Express) وجزء أمامي (React). الدفع مدمج عبر Stripe Checkout.

الميزات الموجودة في النسخة الحالية:
- عرض منتجات (فئات بطاقات وقيم).
- الانتقال إلى Stripe Checkout عند الضغط على شراء.
- تخزين بسيط للطلبات عبر الردود (يمكن توسيعه لاحقًا).

المتطلبات:
- Node.js v16+
- حساب Stripe (لاختبار الدفع يمكنك استخدام مفاتيح الاختبار)
- Git (اختياري)

تشغيل محلي (خطوات سريعة):

1) خلفية (backend)
- افتح مجلد backend
- انسخ ملف .env.example إلى .env وعبّئ القيم:
  - STRIPE_SECRET_KEY: مفتاح Stripe السري (اختباري)
  - STRIPE_PUBLISHABLE_KEY: مفتاح Stripe العام (اختباري) — يستخدم في الواجهة
  - FRONTEND_URL: عنوان الواجهة، مثلاً http://localhost:3000
  - PORT: منفذ السيرفر (مثلاً 4000)
- ثبّت الحزم:
  npm install
- شغّل السيرفر:
  npm run dev
- سيرفر الـ API سيكون على http://localhost:4000

2) واجهة (frontend)
- افتح مجلد frontend
- أنشئ ملف .env.local (أو استخدم طريقة CRA) وضع:
  REACT_APP_STRIPE_PUBLISHABLE_KEY=pk_test_...
  REACT_APP_API_BASE=http://localhost:4000
- ثبّت الحزم:
  npm install
- شغّل الواجهة:
  npm start
- افتح http://localhost:3000

ما الذي يمكنك إضافته لاحقًا (اقتراحات):
- لوحة إدارة لإضافة/حذف/تعديل قيم البطاقات.
- دعم قاعدة بيانات (Postgres/Mongo/SQLite) لتخزين الطلبات والمستخدمين.
- نظام مستخدمين وتوثيق (JWT / OAuth).
- تكامل مع مورد البطاقات الفعلي (API مزود بطاقات) أو رفع أكواد يدوياً.
- Webhooks من Stripe لتأكيد الدفع وتفعيل الشحن (تسليم الكود للمشتري).
- واجهة متعددة اللغات و تصميم جيد (Tailwind/Bootstrap).

ملاحظة أمان:
- استخدم مفاتيح Stripe الآمنة في المتغيرات، ولا تضعها في الكود.
- تأكد من تفعيل HTTPS في بيئة الإنتاج.

إذا أردت، أستطيع الآن:
- إضافة لوحة إدارة بسيطة.
- تعديل الواجهة لتوليد أكواد تلقائية من ملف CSV.
- ربط قاعدة بيانات وتخزين الطلبات.

أخبرني أي وظيفة تريد أن أبنيها بعدها وسأضيفها مباشرة.
